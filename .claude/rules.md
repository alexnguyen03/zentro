# Global Rules

- Always prefer minimal changes over full rewrites
- Never modify unrelated code
- Avoid introducing new dependencies unless necessary
- Follow existing architecture strictly
- If unsure, ask instead of guessing

## Output Rules

- Only show changed code (diff preferred)
- No explanation unless explicitly requested
- Keep responses concise (<200 lines)

## Code Quality

- Avoid magic numbers
- Use clear naming
- Prefer pure functions
- Handle edge cases explicitly
